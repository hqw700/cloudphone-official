# 极速运行指南

为了让开发者可以瞬间预览效果，CloudPhone 提供了开箱即用的一键启动脚本。信令服务器已将静态 Web 资源打包至内部。

---

## ⚡ 三步体验流程

### 第一步：启动信令与 Web 服务

获取解压发布包后，如果是linux或macos在根目录下直接执行以下命令（默认监听端口为 `8443`）：

```bash
# 启动信令服务器并关联 Web-UI 资源 (其中V2为实验性功能，很多功能暂未维护)
./start_server.sh
```

*若为 Windows 用户，请进入 `bin/windows_amd64/` 目录运行 `run_v1.bat` 脚本。*

启动成功后，直接在浏览器中访问：`http://localhost:8443`。

---

### 第二步：配置物理手机环境

1. 准备一台 Android 物理真机，在系统设置中找到并进入「开发者选项」。
2. 开启**USB 调试**
3. 用数据线将手机与电脑连接，并在手机弹出的配对框中选择「允许调试」。

---

### 第三步：部署运行 Agent (宿主机命令行)

进入发布包的 `agentd` 目录，执行以下自动化脚本：

```bash
cd agentd

# 自动探测 CPU 架构并将 Agent 进程推送到手机中运行
# 请将 192.168.1.100 替换为您的信令服务器实际局域网 IP
./run.sh -id dev-phone-01 -signaling ws://192.168.1.100:8443
```
*若为 Windows 用户，暂无提供脚本，参考run.sh手动在adb中执行命令。*
> [!IMPORTANT]
> 执行此脚本不需要您在电脑上安装任何 ADB 工具，脚本内部已内置微型 ADB 协议交互。如果提示端口被占用，请确保电脑上已关闭已存在的后台 `adb server`（可执行 `adb kill-server` 清理）。


### 【可选】如机器中未安装ADB，可以使用网页部署功能

---

## 📂 Web 页面直连

当控制台命令执行成功后，刷新 `http://localhost:8443`，若服务运行在其它机器上，则打开：`http://[服务器IP]:8443`, 您将在仪表盘设备网格中看到名为 `dev-phone-01` 的卡片，点击即可开启低延迟推流和完全的交互操作！
