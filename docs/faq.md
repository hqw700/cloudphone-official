# 常见问题排查 (FAQ)

本章节整理了在局域网多开或公网部署云手机服务时，最常碰到的网络、触控映射及音视频积压故障，并给出了推荐的排错步骤。

---

## 🔍 问题一：Web 页面加载成功但一直显示“黑屏”，提示 WebRTC 连接超时？

### 可能原因：
1. **UDP 端口未开放**：WebRTC 的握手（SDP 交互）是通过 WebSocket 信令通道传输的，即使端口不通，信令阶段依然会显示“连接中”。但是实际推流走的是 UDP 端口。
2. **NAT 协商失败**：如果服务器或客户端身处严苛的对称型 NAT (Symmetric NAT) 网络后，直连 (host/srflx) 收集完毕后依然无法穿透。

### 解决方案：
- **步骤一**：检查宿主机或服务器安全组。确保除了信令监听的端口（如 `8443`）之外，还完全开放了 Agent 绑定的 WebRTC 动态 UDP 端口段。
- **步骤二**：如果在 Docker 内运行，请务必在启动命令中增加 IP 映射和 UDP 单端口映射：
  ```bash
  # 指定外部宿主机的 IP 地址与 UDP 端口
  -external-addr "<宿主机实际公网IP>" -webrtc-port 50000
  ```
- **步骤三**：在 Agent 部署命令中指定有效的 TURN 服务器：
  ```bash
  -ice-servers "turn:username:password@your-turn-ip:3478"
  ```

## 🔍 问题二：网页端拖动时鼠标定位漂移，并且屏幕边缘的点击无法触发？

### 可能原因：
1. **分辨率比例不匹配**：网页投屏容器（HTML Canvas）的拉伸横纵比，与 Android 系统实际的物理像素不符合，导致触控转换坐标时发生了黑边偏移。
2. **虚拟按键遮挡**：部分真机 ROM 底部有隐藏的系统导航栏，导致实际高度探测有误。

### 解决方案：
- **步骤一**：在网页端控制台右上角的设置中，将画面缩放模式调整为「自适应（保持宽高比）」或「1:1 像素映射」，避免强行拉伸。
- **步骤二**：Agent 在初次注册时，会自动调用 `wm size` 获取分辨率。如果发生了长宽颠倒（例如横屏变竖屏），请尝试在控制台手动发送 Shell 指令触发重置：
  ```bash
  wm size reset
  ```

## 🔍 问题三：开启音频通道 (Audio=true) 之后，投屏画面产生长达数秒的严重积压，声音卡顿？

### 可能原因：
1. **音频 PTS 失步**：Android 系统的音频 PTS 并不是按物理墙上时钟流逝，且当手机麦克风/声卡休眠时会产生 PTS 跳跃。桥接端如果强行去对其进行同步，会导致 WebRTC 引擎的 Jitter Buffer 为等待音频帧而恶意积压视频帧。
2. **弱网带宽过载**：开启音频会导致整体带宽增加，如果触发了局域网/公网拥塞，容易引起丢包风暴。

### 解决方案：
- **步骤一**：在 Agent 启动命令行中开启弱网拥塞控制算法（BWE/TWCC）：
  ```bash
  -bwe true
  ```
  这会强制服务器在带宽受限时，自动无延迟热更新 MediaCodec 底层的量化参数（QP）降低视频码率，保障画面的流畅通畅。
- **步骤二**：如果视频积压，可以点击控制台右上角的 **“请求强制 I 帧”** 按钮。Agent 会向物理 scrcpy-server 下发 `18` 关键帧重置指令，强制编码器生成全新带正确时钟的 IDR 帧，瞬间清空 Jitter 缓冲区。
